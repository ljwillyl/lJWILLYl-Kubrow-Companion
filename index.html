How the shared learning works

1. A tester scans and confirms all three known colours.
2. The result is uploaded as pending.
3. You review it in Supabase → Table Editor → kubrow_submissions.
4. Change status to approved.
5. A database trigger copies its three colour slots into the public approved reference library.
6. Every visitor automatically downloads those approved references when the scanner opens. Matching uses CIEDE2000 perceptual colour difference and the nearest approved examples for each colour/slot.

Changing an approved row to rejected or pending removes its reference samples automatically.
No new SQL is required for v7 if v6 is already installed.
